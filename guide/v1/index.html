راهنمای جامع استفاده از کتابخانه آرایش مجازی ARmo

مقدمه

کتابخانه آرایش مجازی ARmo یک راه‌حل پیشرفته برای اعمال آرایش دیجیتال روی تصاویر چهره است. این کتابخانه با استفاده از هوش مصنوعی و تکنولوژی‌های پیشرفته، امکان اعمال آرایش در زمان واقعی را فراهم می‌کند.

نسخه نمایشی آنلاین

برای آشنایی با قابلیت‌های کتابخانه، از نسخه نمایشی آنلاین ما دیدن کنید: https://makeup.armogroup.tech/

4.1. استفاده از callbacks

SDK دو callback مهم onReady و onError را برای مدیریت بهتر وضعیت‌ها ارائه می‌دهد: 

document.addEventListener("DOMContentLoaded", () => {
    // تنظیم توکن
    window.Makeup.token = "YOUR_TOKEN_HERE";

    // ایجاد نمونه از Makeup با callbacks
    const makeupApp = new window.Makeup({
        colors: [
            { code: "01", color: "#000000", url: "path/to/image1.jpg" },
            { code: "02", color: "#4B0082", url: "path/to/image2.jpg" },
            { code: "03", color: "#8B4513" }
        ],
        pattern: "normal",
        face: "lips",
        // callback برای زمانی که SDK آماده استفاده است
        onReady: () => {
            console.log("SDK آماده استفاده است");
            // مخفی کردن loading
            document.getElementById("loading").style.display = "none";
            // نمایش کنترل‌ها
            document.getElementById("controls").style.display = "block";
        },
        // callback برای زمانی که خطایی رخ می‌دهد
        onError: (error) => {
            console.error("خطا در SDK:", error);
            // نمایش پیغام خطا به کاربر
            document.getElementById("error-message").textContent = error.message;
            document.getElementById("error-message").style.display = "block";
        }
    });

    window.makeupApp = makeupApp;
});

نکته: برای استفاده بهتر از callbacks، می‌توانید المان‌های HTML مرتبط را نیز تعریف کنید: 

<div id="armo-makeup-view">
    <!-- نمایش loading -->
    <div id="loading">در حال بارگذاری...</div>

    <!-- نمایش خطا -->
    <div id="error-message" style="display: none; color: red;"></div>

    <!-- کنترل‌های SDK -->
    <div id="controls" style="display: none;">
        <button onclick="makeupApp.changeMakeupType('lips')">رژ لب</button>
        <button onclick="makeupApp.changeMakeupType('eyeshadow')">سایه چشم</button>
        <!-- سایر کنترل‌ها -->
    </div>
</div>

مزایای استفاده از callbacks:

  • مدیریت بهتر وضعیت‌های مختلف SDK
  • نمایش/مخفی کردن خودکار loading
  • نمایش خطاها به کاربر به صورت user-friendly
  • کنترل بهتر زمان نمایش UI و کنترل‌ها
  • امکان اجرای عملیات خاص پس از آماده شدن SDK

  • getAvailableFeatures(): لیست تمام ویژگی‌های فعال را برمی‌گرداند
  • isFeatureEnabled(feature): بررسی می‌کند آیا یک ویژگی خاص فعال است
  • getAvailablePatterns(feature): لیست الگوهای مجاز برای یک ویژگی را برمی‌گرداند

تنظیم شفافیت آرایش

با استفاده از متد setMakeupTransparency می‌توانید میزان شفافیت هر نوع آرایش را تنظیم کنید. مقدار شفافیت باید بین 0 (کاملاً شفاف) تا 1 (کاملاً غلیظ) باشد.

مثال استفاده:

// تنظیم شفافیت رژ لب به 50%
makeupApp.setMakeupTransparency("lips", 0.5);

// تنظیم شفافیت سایه چشم به 70%
makeupApp.setMakeupTransparency("eyeshadow", 0.7);

// تنظیم شفافیت رژگونه به 30%
makeupApp.setMakeupTransparency("blush", 0.3);

2.1 استفاده از callbacks در React Native

import React, { useState } from 'react';
import { View, Text, ActivityIndicator } from 'react-native';
import { WebView } from 'react-native-webview';

const VirtualMakeup = () => {
   const [isReady, setIsReady] = useState(false);
   const [error, setError] = useState(null);

   const htmlContent = `
       <!DOCTYPE html>
       <html>
           <head>
               <link rel="stylesheet" href="https://sdk.armogroup.tech/makeup/v2/makeup.min.css">
               <script src="https://sdk.armogroup.tech/makeup/v2/makeup.js"></script>
           </head>
           <body>
               <div id="armo-makeup-view"></div>
               <script>
                   window.Makeup.token = "YOUR_TOKEN_HERE";
                   const makeup = new window.Makeup({
                       colors: [...],
                       pattern: "normal",
                       face: "lips",
                       onReady: () => {
                           window.ReactNativeWebView.postMessage(JSON.stringify({
                               type: 'ready'
                           }));
                       },
                       onError: (error) => {
                           window.ReactNativeWebView.postMessage(JSON.stringify({
                               type: 'error',
                               message: error.message
                           }));
                       }
                   });
               </script>
           </body>
       </html>
   `;

   const handleMessage = (event) => {
       const data = JSON.parse(event.nativeEvent.data);
       if (data.type === 'ready') {
           setIsReady(true);
       } else if (data.type === 'error') {
           setError(data.message);
       }
   };

   return (
       <View style={{ flex: 1 }}>
           {!isReady && !error && (
               <ActivityIndicator size="large" color="#0000ff" />
           )}
           {error && (
               <Text style={{ color: 'red' }}>{error}</Text>
           )}
           <WebView
               source={{ html: htmlContent }}
               onMessage={handleMessage}
               style={{ flex: 1 }}
           />
       </View>
   );
};

نکته مهم برای React Native: در React Native، ارتباط بین WebView و کد نیتیو از طریق postMessage انجام می‌شود. به همین دلیل، callbacks باید از طریق این مکانیزم پیام‌رسانی عمل کنند.

پیشنهاد: در صورتی که نیاز خاصی ندارید، استفاده از پروفایل‌های آماده (basic یا high) را توصیه می‌کنیم که برای اکثر موارد تنظیمات مناسبی دارند.

ساختار خروجی getTechnicalInfo

{
  version: "v2.0.0", // نسخه SDK
  status: "ready", // وضعیت فعلی SDK
  currentMakeupType: "lips", // نوع آرایش فعلی
  isPremium: false, // وضعیت اکانت
  enabledFeatures: ["lips", "eyeshadow"], // ویژگی‌های فعال
  resolution: { // رزولوشن ویدیو دوربین
    width: 1280,
    height: 720
  },
  browser: { // اطلاعات مرورگر
    userAgent: "Mozilla/5.0 ...",
    isMobile: false
  }
}

توضیحات فیلدها:

  • version: نسخه فعلی SDK
  • status: وضعیت SDK (initializing, loading, ready, error)
  • currentMakeupType: نوع آرایش فعال
  • isPremium: وضعیت پریمیوم اکانت
  • enabledFeatures: لیست تمام ویژگی‌های فعال برای کاربر
  • resolution: تنظیمات رزولوشن دوربین
  • browser: اطلاعات مرورگر و نوع دستگاه

نکته: فیلد enabledFeatures در ورژن 2 اضافه شده و لیست تمام ویژگی‌های فعال را که از طریق توکن مجاز شده‌اند، نمایش می‌دهد.

استفاده از اطلاعات فنی در پروژه:

این اطلاعات می‌تواند برای تنظیمات پیشرفته‌تر، عیب‌یابی، و بهبود تجربه کاربری استفاده شود. برای مثال، می‌توانید با استفاده از isMobile رابط کاربری خاصی برای کاربران موبایل فراهم کنید یا با بررسی status نمایش خطاها یا پیغام‌های آماده‌سازی را مدیریت کنید.

مخفی کردن Color Picker داخلی

اگر می‌خواهید از رنگ‌های سفارشی خودتان استفاده کنید و Color Picker داخلی SDK را مخفی کنید، می‌توانید از گزینه showColorPicker استفاده کنید: 

// ایجاد نمونه از Makeup با Color Picker مخفی
const makeupApp = new window.Makeup({
    colors: [
        { code: "01", color: "#000000" },
        { code: "02", color: "#4B0082" }
    ],
    pattern: "normal",
    face: "lips",
    showColorPicker: false // مخفی کردن Color Picker داخلی
});

// تغییر رنگ از طریق کد
makeupApp.changeMakeupColor("lips", "#FF0000");

اگر گزینه showColorPicker تنظیم نشود، به صورت پیش‌فرض true خواهد بود و Color Picker داخلی نمایش داده می‌شود.

نکته: وقتی showColorPicker: false تنظیم می‌شود، می‌توانید از المان‌های UI خودتان برای تغییر رنگ استفاده کنید و با متد changeMakeupColor رنگ را تغییر دهید.

مثال استفاده با UI سفارشی:

// HTML
<button onclick="changeColor('#FF0000')">قرمز</button>
<button onclick="changeColor('#00FF00')">سبز</button>

// JavaScript
function changeColor(color) {
    makeupApp.changeMakeupColor("lips", color);
}

مدیریت وضعیت SDK

با استفاده از متد getStatus() می‌توانید وضعیت فعلی SDK را دریافت کنید:

  • initializing: در حال راه‌اندازی اولیه
  • loading: در حال بارگذاری منابع
  • ready: آماده استفاده
  • error: خطا در راه‌اندازی

نمونه استفاده از وضعیت‌ها

const status = makeupApp.getStatus();
switch(status) {
   case 'ready':
       // SDK آماده استفاده است
       makeupApp.changeMakeupType("lips");
       break;
   case 'error':
       // نمایش خطا به کاربر
       showError("خطا در راه‌اندازی SDK");
       break;
   default:
       // SDK در حال بارگذاری است
       showLoading();
}

استفاده از المان‌های سفارشی

برای استفاده از المان‌های سفارشی در بخش کنترل‌های SDK، کافیست ابتدا در هنگام راه‌اندازی، گزینه showColorPicker را false قرار دهید: 

const makeup = new window.Makeup({
    showColorPicker: false,
    // سایر تنظیمات...
});

سپس می‌توانید با استفاده از شناسه armo-sdk-custom-controls المان‌های سفارشی خود را تعریف کنید: 

<div id="armo-sdk-custom-controls">
    <button onclick="changeMakeupColor('lips', '#FF0000')">رژلب قرمز</button>
    <button onclick="changeMakeupType('eyeshadow')">سایه چشم</button>
    <!-- هر المان دلخواه دیگر -->
</div>

نکته: المان‌های سفارشی در بخش پایین SDK و در محل کنترل‌ها نمایش داده خواهند شد.

نکات مهم

  • همیشه از معتبر بودن توکن خود اطمینان حاصل کنید.
  • قبل از فراخوانی هر متد، از ready بودن SDK اطمینان حاصل کنید.
  • در محیط‌های کم‌نور به هشدارهای نوری توجه کنید.
  • در موبایل، حتماً مجوزهای دوربین را بررسی کنید.
  • برای بهترین عملکرد، از مرورگرهای به‌روز استفاده کنید.
  • برای استفاده از UI سفارشی می‌توانید Color Picker داخلی را با showColorPicker: false غیرفعال کنید.

سوالات عمومی

1. آیا می‌توانم SDK را به صورت آفلاین استفاده کنم؟

خیر، SDK نیاز به اتصال اینترنت دارد زیرا باید توکن شما به صورت آنلاین اعتبارسنجی شود.

2. چه مرورگرهایی پشتیبانی می‌شوند؟

تمامی مرورگرهای مدرن از جمله:

  • Chrome (نسخه 88 و بالاتر)
  • Firefox (نسخه 85 و بالاتر)
  • Safari (نسخه 14 و بالاتر)
  • Edge (نسخه 88 و بالاتر)

سوالات فنی

3. چرا دوربین کار نمی‌کند؟

موارد زیر را بررسی کنید:

  • مجوز دسترسی به دوربین در مرورگر
  • اتصال HTTPS در محیط وب
  • عدم استفاده همزمان دوربین توسط برنامه‌های دیگر
  • به‌روز بودن مرورگر

4. چگونه می‌توانم کیفیت تشخیص چهره را بهبود دهم؟

برای بهبود تشخیص چهره:

  • از نور کافی و مناسب استفاده کنید
  • مستقیم به دوربین نگاه کنید
  • فاصله مناسب (30-60 سانتیمتر) از دوربین را حفظ کنید
  • از پس‌زمینه ساده استفاده کنید

5. چگونه می‌توانم وضعیت SDK را مانیتور کنم؟

const status = makeupApp.getStatus();
console.log('SDK Status:', status);

10. آیا تصاویر دوربین ذخیره می‌شوند؟

خیر، تمام پردازش‌ها در لحظه و روی دستگاه کاربر انجام می‌شود و هیچ تصویری ذخیره نمی‌شود.

11. توکن من منقضی شده است. چه کار کنم؟

با پشتیبانی تماس بگیرید تا توکن جدید دریافت کنید.

12. چگونه می‌توانم از رنگ‌های سفارشی خودم استفاده کنم؟

دو روش وجود دارد:

  • استفاده از Color Picker داخلی با تعریف آرایه colors
  • مخفی کردن Color Picker داخلی با showColorPicker: false و استفاده از UI سفارشی

13. چگونه می‌توانم از وضعیت SDK در برنامه خود مطلع شوم؟

SDK دو callback اصلی ارائه می‌دهد:

  • onReady: زمانی که SDK کاملاً آماده استفاده است فراخوانی می‌شود
  • onError: در صورت بروز هر گونه خطا فراخوانی می‌شود و پیام خطا را دریافت می‌کند

این callbacks در تمام پلتفرم‌ها (Pure JS، React، React Native) قابل استفاده هستند.
