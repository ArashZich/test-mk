سوالات عمومی

آیا می‌توانم SDK را به صورت آفلاین استفاده کنم؟

خیر، SDK نیاز به اتصال اینترنت دارد زیرا باید:

  • توکن به صورت آنلاین اعتبارسنجی شود
  • منابع مورد نیاز از CDN بارگیری شوند

چه مرورگرهایی پشتیبانی می‌شوند؟

مرورگرهای مدرن با نسخه‌های زیر پشتیبانی می‌شوند:

  • Chrome نسخه 88 و بالاتر
  • Firefox نسخه 85 و بالاتر
  • Safari نسخه 14 و بالاتر
  • Edge نسخه 88 و بالاتر

سوالات فنی

چرا دوربین کار نمی‌کند؟

موارد زیر را بررسی کنید:

  • مجوز دسترسی به دوربین در مرورگر
  • اتصال HTTPS در محیط وب
  • عدم استفاده همزمان دوربین توسط برنامه‌های دیگر
// بررسی دسترسی به دوربین
if (!navigator.mediaDevices || !navigator.mediaDevices.getUserMedia) {
    console.error("دوربین در این مرورگر پشتیبانی نمی‌شود");
}

چگونه می‌توانم کیفیت تشخیص چهره را بهبود دهم؟

  1. از نور کافی و مناسب استفاده کنید
  2. مستقیم به دوربین نگاه کنید
  3. فاصله مناسب (30-60 سانتیمتر) را حفظ کنید
  4. از پس‌زمینه ساده استفاده کنید

چه تفاوتی بین استایل‌های مختلف خط چشم وجود دارد؟

SDK آرایش مجازی دو استایل متفاوت برای خط چشم ارائه می‌دهد:

  • normal: خط چشم استاندارد که در امتداد مژه‌های بالایی کشیده می‌شود.
  • lashed: خط چشم ضخیم‌تر با افکت مژه‌های بلند که ظاهری شبیه به مژه‌های مصنوعی ایجاد می‌کند. 
// تنظیم استایل خط چشم
makeupApp.changeMakeupType("eyeliner");
makeupApp.setMakeupPattern("eyeliner", "lashed");

// تنظیم ضخامت خط چشم
makeupApp.setMakeupTransparency("eyeliner", 0.7);

چگونه وضعیت SDK را مانیتور کنم؟

// روش 1: استفاده از getStatus
const status = makeupApp.getStatus();

// روش 2: استفاده از callbacks
const makeup = new window.Makeup({
    onReady: () => console.log("SDK آماده است"),
    onError: (error) => console.error("خطا:", error)
});

سوالات پیاده‌سازی

چگونه می‌توانم از رنگ‌های سفارشی استفاده کنم؟

دو روش وجود دارد:

// روش 1: تعریف رنگ‌ها در زمان راه‌اندازی
const makeup = new window.Makeup({
    colors: [
        { code: "01", color: "#FF0000" },
        { code: "02", color: "#00FF00", url: "path/to/image.jpg" }
    ]
});

// روش 2: تغییر مستقیم رنگ
makeupApp.changeMakeupColor("lips", "#FF0000");

چگونه لودینگ سفارشی تعریف کنم؟

با استفاده از المان با ID خاص:

<div id="armo-sdk-custom-loading">
    <div class="loading-spinner"></div>
    <p>در حال بارگذاری...</p>
</div>

سوالات استخراج رنگ از تصاویر

آیا می‌توانم از تصاویر با فرمت‌های مختلف برای استخراج رنگ استفاده کنم؟

بله، SDK از فرمت‌های استاندارد تصویر مانند JPG، PNG و WebP پشتیبانی می‌کند. برای نتایج بهتر، تصاویر با کیفیت بالا و با زمینه ساده توصیه می‌شود.

چه محدودیت‌هایی برای استخراج رنگ از تصاویر وجود دارد؟

  • تصویر باید دسترسی CORS داشته باشد
  • تصاویر خیلی بزرگ ممکن است زمان پردازش بیشتری نیاز داشته باشند
  • کیفیت پایین تصویر می‌تواند بر دقت استخراج رنگ تأثیر منفی بگذارد
  • پس‌زمینه‌های پیچیده ممکن است رنگ استخراج شده را منحرف کنند

آیا می‌توانم رنگ‌های متعددی را از یک تصویر استخراج کنم؟

در حال حاضر، SDK فقط رنگ غالب را از هر تصویر استخراج می‌کند. برای استخراج چند رنگ مختلف، می‌توانید از تصاویر متفاوت استفاده کنید یا رنگ‌ها را به صورت دستی تعیین کنید.

اگر استخراج رنگ با خطا مواجه شود، چه اتفاقی می‌افتد؟

در صورت بروز خطا در استخراج رنگ (مثلاً به دلیل مشکلات CORS، تصویر نامعتبر، یا تصویر بسیار کوچک)، SDK به طور خودکار از رنگ خاکستری (#808080) به عنوان پشتیبان استفاده می‌کند. همچنین خطا در کنسول مرورگر لاگ می‌شود. 

// مدیریت خطا در استخراج رنگ
makeupApp.addColorFromImage("https://example.com/invalid-image.jpg", "ERR_CODE")
    .then(color => {
        console.log(`رنگ استخراج شده یا پشتیبان: ${color}`);
    })
    .catch(error => {
        // در عمل این بلاک اجرا نمی‌شود زیرا SDK خودش خطا را مدیریت می‌کند
        // اما برای اطمینان می‌توان آن را اضافه کرد
        console.error("خطا در استخراج رنگ:", error);
    });

عیب‌یابی

پیام "توکن نامعتبر است" دریافت می‌کنم

  1. از معتبر بودن توکن اطمینان حاصل کنید
  2. تاریخ انقضای توکن را بررسی کنید
  3. توکن را قبل از ایجاد نمونه SDK تنظیم کنید: 
    window.Makeup.token = "YOUR_TOKEN";
const makeup = new window.Makeup(...);

در موبایل تصویر وارونه نمایش داده می‌شود

این مشکل با افزودن CSS زیر برطرف می‌شود:

.armo-sdk-input-video {
    transform: scaleX(-1);
}

خطای CORS هنگام استخراج رنگ از تصویر دریافت می‌کنم

این خطا زمانی رخ می‌دهد که سرور میزبان تصویر، اجازه دسترسی به منابع خود را از دامنه شما نمی‌دهد. برای رفع این مشکل:

  1. تصاویر را در همان دامنه وب‌سایت خود میزبانی کنید
  2. از سرویس‌هایی استفاده کنید که هدرهای CORS مناسب را ارائه می‌دهند
  3. از یک پروکسی CORS در سمت سرور استفاده کنید
  4. با مالک سرویس میزبانی تصویر تماس بگیرید و درخواست کنید هدرهای CORS مناسب را تنظیم کند
